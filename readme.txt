3.设计思路：
因为是股票交易系统，所以设计原则应该是支持多并发，股市持仓行情也需要实时更新，所以需要持仓存入内存实时计算，
更新持仓时需要对当前股票的持仓加锁再进行计算。

并发的场景下，高优先级的指令可能会延迟到达，所以设计了排序列表，支持延迟交易排队机制，
待交易到达后在对持仓量进行一次合并计算，确保持仓的正确性。
